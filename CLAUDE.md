# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
pnpm dev            # Start dev server at localhost:3000
pnpm lint           # ESLint
pnpm lint:fix       # ESLint with auto-fix
pnpm format:check   # Prettier check
pnpm format         # Prettier write
pnpm typecheck      # pnpm typegen && tsc --noEmit
pnpm test:run       # Vitest single run
pnpm test           # Vitest watch mode
pnpm build          # Production build (standalone output)
```

Run a single test file:

```bash
pnpm test:run test/regions.test.ts
```

Full quality gate before handing off source, config, or test changes:

```bash
pnpm lint && pnpm format:check && pnpm typecheck && pnpm test:run && pnpm build
```

Use `pnpm` only — no npm, Yarn, or Bun.

## Architecture

Next.js 16 App Router campaign microsite for Jesus Film Project's World Cup 2026 activation. One homepage and one page per activation region, each available in 12 locales.

### Routing

- `proxy.ts` is the Next.js middleware. It handles locale detection and redirect, strips `/en` prefixes (English is canonical without locale prefix), and normalizes region aliases (e.g. `/Europe` → `/europe`, `/EA` → `/east-asia`) to canonical IDs via `getRegion()`.
- All page routes live under `app/[locale]/` — the `[locale]` segment is present in the filesystem but English routes are rewritten to look prefix-free to users.
- `i18n/routing.ts` owns the supported locale list and per-locale metadata (direction, OG locale, labels).

### Region configuration

`lib/regions.ts` is the single source of truth for activation regions. Each `Region` has:

- `id` — canonical URL segment (e.g. `"east-asia"`)
- `code` — JSONBin region label used to filter country-view rows
- `teamId` — Jesus Film GraphQL API team ID for fetching journeys
- `getRegion(id)` — fuzzy lookup supporting aliases; `getRegionById(id)` — exact match

### Data fetching

- **Journeys** (`lib/journeys.ts`): fetches from Jesus Film GraphQL API by `teamId`, cached 1 hour. Throws on failure; region pages catch and continue with empty list.
- **Country views** (`lib/country-views.ts`): fetches public JSONBin payload, cached 1 hour. Returns unavailable state (no throw) on failure. Filters aggregate rows and normalizes country identity via `lib/country-display.ts` + `Intl.DisplayNames`.

### Internationalization

- `messages/en.json` is the canonical key structure. All other `messages/*.json` must match it exactly — `test/messages.test.ts` enforces parity.
- Server components use `getTranslations`; client components use `useTranslations`.
- Internal links must use locale-aware navigation from `i18n/navigation.ts`.
- Arabic (`ar`) and Urdu (`ur`) are RTL — use Tailwind logical utilities (`ms`, `me`, `ps`, `pe`, `start`, `end`) for layout-sensitive styles.
- Do not add user-facing strings directly in components. Always add to `messages/en.json` first.

### SEO and metadata

- `lib/site.ts` owns `SITE_URL` and metadata defaults. Keep it aligned with the production domain — sitemap, robots, structured data, and OG images all derive from it.
- Social images are generated by `app/[locale]/opengraph-image.tsx` and `app/[locale]/twitter-image.tsx`.
- JSON-LD is emitted in `app/[locale]/layout.tsx` (Organization/WebSite) and `app/[locale]/[id]/page.tsx` (WebPage/Breadcrumb).

### Deployment

Deployed to Railway via Railpack. `next.config.ts` sets `output: "standalone"`. A `postbuild` script copies `public/` and `.next/static/` into `.next/standalone/` so the standalone server can serve assets. There is intentionally no Dockerfile.
